SQL Data Types for MySQL, SQL Server, and MS Access

The data type of a column defines what value the column can hold: integer, character, money, date and time, binary, and more.

Overview

Each column in a table requires a name and a data type. The SQL developer chooses the type of data stored, which affects storage, validation, and behavior. Note: data types differ across databases.

MySQL Data Types

String

  • CHAR(size): Fixed length string, 0-255 characters.
  • VARCHAR(size): Variable length string, up to 65535 characters.
  • BINARY(size): Fixed-length binary string.
  • VARBINARY(size): Variable-length binary string.
  • TINYBLOB / TINYTEXT: Max length 255 bytes / chars.
  • TEXT / BLOB: Max length 65535 bytes / chars.
  • MEDIUMTEXT / MEDIUMBLOB: Max length 16,777,215 bytes / chars.
  • LONGTEXT / LONGBLOB: Max length 4,294,967,295 bytes / chars.
  • ENUM / SET: Predefined lists of values.

Numeric

  • INT / INTEGER / BIGINT / SMALLINT / MEDIUMINT / TINYINT: Integer types with various ranges.
  • FLOAT / DOUBLE / DECIMAL: Floating-point and exact numeric types.
  • BIT: Bit values, 1–64 bits.
  • BOOL / BOOLEAN: True/False values.

Date and Time

  • DATE: YYYY-MM-DD
  • DATETIME / TIMESTAMP: Date and time values
  • TIME: Time only
  • YEAR: Four-digit year

SQL Server Data Types

  • String: char(n), varchar(n), varchar(max), text, nchar, nvarchar(n), nvarchar(max), ntext
  • Numeric: bit, tinyint, smallint, int, bigint, decimal(p,s), numeric(p,s), smallmoney, money, float(n), real
  • Date/Time: datetime, datetime2, smalldatetime, date, time, datetimeoffset, timestamp
  • Other: sql_variant, uniqueidentifier, xml, cursor, table

MS Access Data Types

  • Text, Memo, Byte, Integer, Long, Single, Double, Currency, AutoNumber, Date/Time, Yes/No, Ole Object, Hyperlink, Lookup Wizard
